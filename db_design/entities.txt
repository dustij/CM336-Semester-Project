USER
  Represents the person using the app
  - id (PK)
  - email
  - password_hash
  - display_name

MESOCYCLE
  The reusable workout plan template
  - id (PK)
  - user_id (FK)
  - title
  - description
  - duration_weeks
  - is_favorite

MESOCYCLE_DAY
  A planned training day within the mesocycle
  - id (PK)
  - mesocycle_id (FK)
  - name
  - day_of_week
  - day_order

PLANNED_EXERCISE
  The planned exercise inside a mesocycle day
  - id (PK)
  - mesocyle_day_id (FK1)
  - exercise_id (FK2)
  - exercise_order

EXERCISE
  A catalog exercise
  - id (PK)
  - name
  - description
  - exercise_type
  - muscle_group
  - equipment
  - is_custom
  - created_by_user_id (FK) nullable
  - difficulty_level

MESOCYCLE_INSTANCE
  One actual run of a mesocycle
  - id (PK)
  - mesocyle_id (FK1)
  - user_id (FK2)
  - start_date
  - end_date nullable
  - status

WORKOUT_SESSION
  One actual completed or started session tied to a mesocycle instance
  - id (PK)
  - mesocycle_instance_id (FK1)
  - mesocycle_day_id (FK2)
  - week_number
  - session_date
  - start_time nullable
  - end_time nullable
  - status 

PERFORMED_EXERCISE
  The actual exercise performed during a specific session
  - id (PK)
  - workout_session_id (FK1)
  - planned_exercise_id (FK2) nullable
  - exercise_id (FK3)
  - exercise_order
  - is_substituted
  - completed
  - notes

  note: This lets us handle exercises from the plan or exercises added on the fly

PERFORMED_SET
  The most granular history record, belongs to one performed_exercise
  - id (PK)
  - performed_exercise_id (FK)
  - set_number
  - weight
  - reps
  - completed
